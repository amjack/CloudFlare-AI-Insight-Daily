## 🔍 Cron 表达式说明

Cron 表达式格式：`分 时 日 月 星期`

| 字段 | 允许值 | 说明 |
|------|--------|------|
| 分钟 | 0-59 | 第几分钟 |
| 小时 | 0-23 | 第几小时 |
| 日期 | 1-31 | 每月第几天 |
| 月份 | 1-12 | 第几个月 |
| 星期 | 0-7 | 星期几（0和7都表示周日）|

### 当前配置

**GitHub Actions：** `0 1 * * *`
- 0 = 第0分钟
- 1 = 第1小时（UTC）
- `* * *` = 每天、每月、每周

**Docker：** `0 9 * * *`
- 0 = 第0分钟
- 9 = 第9小时（北京时间）
- `* * *` = 每天、每月、每周

## 🔄 如何修改为其他时间

如果需要修改为其他执行时间：

### 北京时间早上 10:00

**GitHub Actions：**
```yaml
schedule:
  - cron: '0 2 * * *'  # UTC 02:00 = 北京时间 10:00
```

**Docker：**
```dockerfile
RUN echo "0 10 * * * /app/scripts/build.sh ..." > /etc/crontabs/root
```

### 北京时间晚上 20:00

**GitHub Actions：**
```yaml
schedule:
  - cron: '0 12 * * *'  # UTC 12:00 = 北京时间 20:00
```

**Docker：**
```dockerfile
RUN echo "0 20 * * * /app/scripts/build.sh ..." > /etc/crontabs/root
```

### 每天两次：早上9点 + 晚上9点

**GitHub Actions：**
```yaml
schedule:
  - cron: '0 1 * * *'   # 北京时间 09:00
  - cron: '0 13 * * *'  # 北京时间 21:00
```

**Docker：**
```dockerfile
# 两行 cron 任务
RUN echo "0 9 * * * /app/scripts/build.sh ..." > /etc/crontabs/root && \
    echo "0 21 * * * /app/scripts/build.sh ..." >> /etc/crontabs/root && \
    chmod 600 /etc/crontabs/root
```

### 2. 夏令时

中国不使用夏令时，但如果你在使用夏令时的地区部署，需要考虑时差变化。

### 3. 立即生效

- **GitHub Actions**：提交代码后立即生效
- **Docker**：需要重新构建镜像并重启容器

### 4. 日志查看

- **GitHub Actions**：在 `Actions` 标签页查看执行历史
- **Docker**：使用 `docker logs ai-daily-cron` 查看日志
