# 测试 mdBook 本地构建

## 问题
工作流执行后，gh-pages 分支依然没有 index.html

## 可能的原因

1. **book.toml 配置问题**
   - `src = "."` 可能导致 mdBook 构建混乱
   - 当前目录有太多非 mdBook 文件

2. **SUMMARY.md 格式问题**
   - 虽然格式看起来正确，但可能 mdBook 没有识别

3. **README.md 内容问题**
   - 可能 README.md 格式有问题

## 诊断命令

在 GitHub Actions 工作流日志中查找：

```
Build mdBook 步骤的输出
```

应该看到类似：
```
[INFO] (mdbook::book): Book building has started
[INFO] (mdbook::book): Running the html backend
[INFO] (mdbook::renderer): Generating output for html backend
```

## 解决方案

### 方案 1：使用标准 src 目录（推荐）

创建标准的 mdBook 目录结构：

```bash
cd /Applications/zhangjunjie/github/CloudFlare-AI-Insight-Daily

# 1. 创建 docs 目录
mkdir -p docs

# 2. 移动 mdBook 源文件到 docs 目录
cp README.md docs/
cp SUMMARY.md docs/
cp -r daily docs/
cp -r today docs/
cp -r podcast docs/

# 3. 修改 book.toml
# 将 src = "." 改为 src = "docs"
```

### 方案 2：添加 .nojekyll 到源目录

mdBook 可能需要特殊配置。

### 方案 3：简化 SUMMARY.md

创建一个最简单的 SUMMARY.md 测试：

```markdown
# Summary

- [首页](README.md)
```

## 需要检查的信息

1. **工作流日志**
   - Build mdBook 步骤是否成功
   - 是否有错误或警告信息

2. **book 目录内容**
   - 工作流构建后，book 目录是否有 index.html
   - 如果本地有，但 gh-pages 没有，说明是部署问题

3. **gh-pages 分支内容**
   - 是否有其他 HTML 文件
   - 文件的时间戳是否更新
