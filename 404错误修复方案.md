# 🚨 404 错误修复方案

## 问题诊断结果

### ❌ 发现的问题

1. **gh-pages 分支没有 index.html**
   ```bash
   # gh-pages 分支文件列表
   ✅ 404.html
   ✅ print.html
   ✅ toc.html
   ✅ css/, fonts/ 等
   ❌ index.html  ← 缺少首页！
   ```

2. **本地 book 目录也没有 index.html**
   - 说明 mdBook 构建不完整

### 🔍 原因分析

- mdBook 构建失败或不完整
- 部署的是不完整的 book 目录
- 缺少首页导致访问根路径时出现 404

---

## 🎯 解决方案

### 方案 1：删除旧 book 目录，重新构建（推荐）

执行以下命令：

```bash
cd /Applications/zhangjunjie/github/CloudFlare-AI-Insight-Daily

# 1. 删除旧的 book 目录
rm -rf book

# 2. 提交删除
git add book
git commit -m "删除旧的不完整 book 目录"
git push origin main

# 3. 手动触发工作流重新构建
echo "现在去 GitHub Actions 手动触发工作流"
echo "https://github.com/amjack/CloudFlare-AI-Insight-Daily/actions"
```

**原理：**
- 删除旧的不完整 book 目录
- 让 GitHub Actions 重新构建完整的 book 目录
- 重新部署到 gh-pages 分支

---

### 方案 2：修改 book.toml 配置

可能是 `src = "."` 配置有问题，改用标准配置：

```toml
[book]
authors = ["张褚"]
language = "zh"
src = "src"           # 改为标准的 src 目录
title = "AI 日刊 - By 张褚"

[output.html]
git-repository-url = "https://github.com/amjack/CloudFlare-AI-Insight-Daily"
site-url = "/CloudFlare-AI-Insight-Daily/"
default-theme = "light"
preferred-dark-theme = "navy"
```

然后创建标准目录结构：

```bash
cd /Applications/zhangjunjie/github/CloudFlare-AI-Insight-Daily

# 创建 src 目录
mkdir -p src

# 移动文件到 src 目录
mv SUMMARY.md src/
mv README.md src/
mv daily src/
mv today src/
mv podcast src/

# 提交修改
git add .
git commit -m "重构为标准 mdBook 目录结构"
git push origin main
```

## ⚠️ 常见问题

### Q1: 删除 book 目录会影响网站吗？

**答：** 不会。gh-pages 分支是独立的，删除本地 book 目录不影响已部署的网站。

### Q2: 为什么工作流构建的 book 目录没有 index.html？

**可能原因：**
1. SUMMARY.md 配置问题
2. README.md 内容问题
3. book.toml 配置问题
4. mdBook 版本问题

### Q3: 如果重新构建后还是 404？

**检查步骤：**
1. 查看 Actions 工作流日志，特别是 "Build mdBook" 步骤
2. 确认 gh-pages 分支有 index.html
3. 确认 GitHub Pages 设置正确：
   - Branch: gh-pages
   - Folder: / (root)
4. 清除浏览器缓存
5. 等待 5-10 分钟（GitHub Pages 更新有延迟）
