# book.toml 配置错误修复说明

## 🐛 错误信息

```
ERROR Invalid configuration file

Caused by: TOML parse error at line 6, column 1
  |
6 | create-missing = true  
  | ^^^^^^^^^^^^^^
unknown field `create-missing`, expected one of `title`, `authors`, 
`description`, `src`, `language`, `text-direction`
```

## ❌ 问题原因

`create-missing = true` 不是 mdbook 的有效配置项。

这个字段可能是从其他项目复制过来的，或者是旧版本的配置项。

## ✅ 已修复

### 修改前的 book.toml

```toml
[book]
authors = []
language = "zh"
src = "."
title = "AI 日刊 - By 张褚"
create-missing = true  # ❌ 无效字段

[output.html]
git-repository-url = "https://github.com/amjack/CloudFlare-AI-Insight-Daily"
default-theme = "light"
preferred-dark-theme = "navy"
```

### 修改后的 book.toml

```toml
[book]
authors = ["张褚"]
language = "zh"
src = "."
title = "AI 日刊 - By 张褚"
# ✅ 移除了 create-missing

[output.html]
git-repository-url = "https://github.com/amjack/CloudFlare-AI-Insight-Daily"
default-theme = "light"
preferred-dark-theme = "navy"
```

## 📝 mdbook 有效的配置项

### [book] 部分

| 配置项 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| `title` | String | 书籍标题 | `"AI 日刊"` |
| `authors` | Array | 作者列表 | `["张褚"]` |
| `description` | String | 书籍描述 | `"每日 AI 资讯"` |
| `src` | String | 源文件目录 | `"."` 或 `"src"` |
| `language` | String | 语言代码 | `"zh"` 或 `"en"` |
| `text-direction` | String | 文字方向 | `"ltr"` 或 `"rtl"` |

### [output.html] 部分

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `theme` | String | 默认主题 |
| `default-theme` | String | 默认主题 |
| `preferred-dark-theme` | String | 暗色主题 |
| `git-repository-url` | String | GitHub 仓库地址 |
| `git-repository-icon` | String | 仓库图标 |

## 🔍 其他错误信息说明

### "在目录 'daily' 中没有找到 'YYYY-MM-DD.md' 格式的文件"

**说明：** 这是正常的，表示 daily 目录还没有日刊内容。

**解决：** 
- 当 Cloudflare Workers 生成日刊后，会创建类似 `2025-11-28.md` 的文件
- 或者手动创建测试文件：
  ```bash
  echo "# 2025-11-28 AI 日刊" > daily/2025-11-28.md
  ```

### "SUMMARY.md 已在 '.' (或当前目录) 中生成"

**说明：** 生成脚本运行成功，但因为 daily 目录为空，所以只生成了基本的 SUMMARY.md

**这是正常的！** 不影响 mdbook 构建。
