## 🔍 技术说明

### 问题原因

GitHub Pages 将网站部署在子路径：
```
https://amjack.github.io/CloudFlare-AI-Insight-Daily/
                         └─────────────┬────────────────┘
                                    子路径
```

mdBook 默认生成的 HTML 中，资源路径是相对路径：
```html
<!-- 错误：无法加载 -->
<link rel="stylesheet" href="/css/chrome.css">

<!-- 正确：加上子路径前缀 -->
<link rel="stylesheet" href="/CloudFlare-AI-Insight-Daily/css/chrome.css">
```

### 解决方案

在 `book.toml` 中配置 `site-url`：
```toml
[output.html]
site-url = "/CloudFlare-AI-Insight-Daily/"
```

mdBook 会自动为所有资源路径添加这个前缀。

## 🐛 如果样式还是不正常

### 1. 清除缓存
**最常见的原因！**

- 按 `Ctrl + Shift + Delete`（或 Mac: `Cmd + Shift + Delete`）
- 选择"缓存的图像和文件"
- 清除最近 1 小时或全部
- 或者使用无痕模式访问

### 2. 检查工作流
```
https://github.com/amjack/CloudFlare-AI-Insight-Daily/actions
```
- 确认工作流执行成功
- 查看 "Build mdBook" 步骤的日志
- 确认没有错误信息

### 3. 检查 gh-pages 分支
```
https://github.com/amjack/CloudFlare-AI-Insight-Daily/tree/gh-pages
```
- 确认 gh-pages 分支有更新
- 查看最近的提交时间
- 确认有 css/ 和 fonts/ 等文件夹

### 4. 使用浏览器开发者工具
1. 打开网站
2. 按 F12 打开开发者工具
3. 切换到 Network 标签
4. 刷新页面
5. 查看是否有 404 错误
6. 检查 CSS 文件的 URL 是否正确

**正确的 CSS URL 应该是：**
```
https://amjack.github.io/CloudFlare-AI-Insight-Daily/css/chrome.css
```

**错误的 URL 是：**
```
https://amjack.github.io/css/chrome.css （缺少仓库名）
```

## 📚 额外优化（可选）

### 自定义主题颜色

编辑 `book.toml`，添加：
```toml
[output.html]
additional-css = ["theme/custom.css"]
```

然后创建 `theme/custom.css`：
```css
/* 自定义样式 */
.sidebar {
    background-color: #f5f5f5;
}

h1 {
    color: #2c3e50;
}
```

### 添加网站图标

将 `favicon.png` 放在项目根目录，在 `book.toml` 中配置：
```toml
[output.html]
favicon = "favicon.png"
```
