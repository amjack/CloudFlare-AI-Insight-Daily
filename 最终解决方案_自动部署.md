## ⚙️ 工作流修改说明

### 修改前（旧方案）：
```yaml
- name: Commit and push mdBook output
  run: |
    git add -f book/
    git commit -m "自动构建 mdBook 网站 [skip ci]"
    git push origin main
```
**问题：** 
- 需要提交 book 到 main 分支
- 污染 main 分支历史
- GitHub 有时识别不到 /book 目录

### 修改后（新方案）：
```yaml
- name: Deploy to GitHub Pages
  uses: peaceiris/actions-gh-pages@v3
  with:
    github_token: ${{ secrets.GITHUB_TOKEN }}
    publish_dir: ./book
    publish_branch: gh-pages
```
**优势：**
- ✅ 自动创建 gh-pages 分支
- ✅ 自动部署到 GitHub Pages
- ✅ 不污染 main 分支
- ✅ 100% 可靠

## ❓ 常见问题

### Q1: gh-pages 分支会自动创建吗？

**答：** 是的！工作流首次运行成功后会自动创建 gh-pages 分支。

### Q2: 需要删除 main 分支的 book 目录吗？

**答：** 可以删除也可以保留，不影响 GitHub Pages 部署。如果想删除：
```bash
git rm -rf book
git commit -m "清理 book 目录"
git push origin main
```

### Q3: 工作流执行失败怎么办？

**常见原因：**
1. **权限问题** - 检查仓库 Settings → Actions → General → Workflow permissions
   - 选择 "Read and write permissions"
2. **GITHUB_TOKEN 问题** - 这个 token 是自动提供的，不需要配置

### Q4: 网站还是 404？

**解决步骤：**
1. 确认 gh-pages 分支已创建
2. 确认 Pages 设置选择了 gh-pages 分支
3. 等待 5 分钟（初次部署需要时间）
4. 清除浏览器缓存
