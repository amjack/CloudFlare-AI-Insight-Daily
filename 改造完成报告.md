## 🚀 快速开始

### 方案一：GitHub Actions（推荐）

#### 步骤 1：获取 Webhook URL

以**钉钉**为例：
1. 打开钉钉群聊 → 群设置 → 智能群助手 → 添加机器人
2. 选择"自定义"机器人
3. 安全设置选择"自定义关键词"，添加：`AI日刊`
4. 复制 Webhook 地址

#### 步骤 2：配置 GitHub

1. 打开仓库 → `Settings` → `Secrets and variables` → `Actions`

2. 添加 **Secret**：
   - Name: `WEBHOOK_URL`
   - Value: 粘贴你的 webhook 地址

3. 添加 **Variable**（切换到 Variables 标签）：
   - Name: `WEBHOOK_TYPE`
   - Value: `dingtalk`

4. （可选）添加自定义 GitHub Pages URL：
   - Name: `GITHUB_PAGES_URL`
   - Value: `https://your-username.github.io/your-repo`

#### 步骤 3：测试

1. 进入 `Actions` → 选择 `Build Daily Journal`
2. 点击 `Run workflow`
3. 等待执行完成
4. 检查钉钉群是否收到通知

---

### 方案二：Docker 部署

#### 修改 Dockerfile

编辑 `cron-docker/Dockerfile`，取消注释并修改：

```dockerfile
# Webhook 通知配置
ENV WEBHOOK_URL="https://your-webhook-url"
ENV WEBHOOK_TYPE="dingtalk"
ENV GITHUB_PAGES_URL="https://your-username.github.io/your-repo"
```

#### 运行容器

```bash
cd cron-docker
docker build -t ai-daily-cron-job .
docker run -d \
  --name ai-daily-cron \
  -p 4399:4399 \
  -e WEBHOOK_URL="你的webhook地址" \
  -e WEBHOOK_TYPE="dingtalk" \
  -e GITHUB_PAGES_URL="https://your-pages-url" \
  --restart always \
  ai-daily-cron-job
```

#### 测试

```bash
# 进入容器
docker exec -it ai-daily-cron /bin/sh

# 手动执行构建脚本
/app/scripts/build.sh /app/scripts/work

# 或直接测试 webhook
WEBHOOK_URL="你的webhook地址" \
WEBHOOK_TYPE="dingtalk" \
GITHUB_PAGES_URL="https://test.com" \
/app/scripts/work/webhook.sh
```

## ⚠️ 注意事项

### 1. GitHub Pages 访问地址

默认格式为：`https://<username>.github.io/<repo-name>`

如果你自定义了域名，需要配置 `GITHUB_PAGES_URL` 变量。

### 2. 安全设置

**重要：** 不要将 webhook URL 提交到代码仓库！

- GitHub Actions：使用 Secrets
- Docker：使用环境变量或构建参数

### 3. 钉钉关键词

如果使用钉钉，必须在机器人安全设置中添加关键词 `AI日刊`，否则无法接收通知。

### 4. 网络访问

确保 GitHub Actions 或 Docker 容器能够访问你的 webhook URL。

### 5. 测试建议

建议先在测试群或测试频道配置 webhook，确认正常后再配置到正式环境。

## 🐛 常见问题

### Q1: 没有收到通知？

**检查清单：**
- ✅ 已配置 `WEBHOOK_URL`
- ✅ `WEBHOOK_TYPE` 设置正确
- ✅ Webhook URL 未过期
- ✅ 钉钉关键词已设置
- ✅ 查看 GitHub Actions 日志

### Q2: 如何自定义通知内容？

编辑 `.github/workflows/build-daily-book.yml`：

```yaml
MESSAGE="${TODAY_DATE} AI日刊，地址：${PAGES_URL}"
```

修改为你想要的格式。

### Q3: 支持同时发送到多个平台吗？

目前不支持。如有需求，可以：
- 在工作流中复制通知步骤
- 使用 webhook 中转服务

### Q4: 如何关闭通知？

删除 `WEBHOOK_URL` Secret 即可，工作流会自动跳过通知步骤。

## 💡 技术实现

### 架构设计

```
GitHub Actions / Docker Cron
    ↓
构建 mdBook 静态站点
    ↓
部署到 GitHub Pages
    ↓
调用 webhook.sh 脚本
    ↓
发送 HTTP POST 请求
    ↓
通知平台（钉钉/企业微信/飞书等）
```

### 核心代码

**webhook.sh 脚本结构：**
```bash
1. 参数验证（WEBHOOK_URL, WEBHOOK_TYPE, GITHUB_PAGES_URL）
2. 日期获取（支持参数或默认今天）
3. 消息构造（格式：YYYY-MM-DD AI日刊，地址：URL）
4. 平台判断（根据 WEBHOOK_TYPE）
5. HTTP 请求（curl POST JSON）
6. 结果处理（成功/失败）
```

**GitHub Actions 集成：**
```yaml
- name: Send Webhook Notification
  if: success()
  run: |
    # 检查配置
    # 获取日期和 URL
    # 根据 WEBHOOK_TYPE 发送请求
```
